# 环境变量说明与收敛策略

## 背景

`.env-example` 过去同时承担了 3 个职责：

1. 展示最小可运行配置
2. 暴露全部高级调优项
3. 记录历史兼容变量

这会导致首次配置时信息量过大，也容易让人误以为所有变量都必须填写。

本次整理的目标不是删除能力，而是做两件事：

1. 把高频、关键、必须关注的配置留在主视图
2. 把有默认值、可共享、仅少数场景才会改的配置收进文档

## 收敛结论

### 已合并或收口的配置

#### 1. WebSocket 地址支持共享基础地址

新增：

- `POLYMARKET_WS_BASE_URL`

它用于统一派生两个通道地址：

- `CLOB_WS_URL=<base>/market`
- `USER_WS_URL=<base>/user`

优先级如下：

1. `CLOB_WS_URL` / `USER_WS_URL`
2. `POLYMARKET_WS_BASE_URL`
3. 官方默认值 `wss://ws-subscriptions-clob.polymarket.com/ws`

适用场景：

- 你只是把官方 WebSocket 网关整体切到自建反代
- 你希望 `.env` 只维护一个共享地址，而不是分别维护两个通道 URL

#### 2. WebSocket 重连间隔支持共享默认值

新增：

- `POLYMARKET_WS_RECONNECT_MS`

它会同时作为下面两个变量的共享默认值：

- `MARKET_WS_RECONNECT_MS`
- `USER_WS_RECONNECT_MS`

优先级如下：

1. 通道级变量
2. `POLYMARKET_WS_RECONNECT_MS`
3. 默认值 `1000`

适用场景：

- 市场通道和用户通道都希望使用同一重连节奏
- 只有在两个通道重连策略明显不同的情况下，才需要分别配置

#### 3. 官方基础设施地址不再要求写入 live 配置

下面这些变量现在都有稳定默认值，因此不再需要在 `live` 模式下强制填写：

- `CLOB_HTTP_URL`
- `POLYMARKET_WS_BASE_URL`
- `USER_WS_URL`
- `RPC_URL`
- `USDC_CONTRACT_ADDRESS`
- `POLYMARKET_RELAYER_URL`
- `POLYMARKET_CTF_CONTRACT_ADDRESS`

这类变量只应在下面情况出现于 `.env`：

- 你使用自建 RPC / 网关 / 反代
- 你需要切到非默认 relayer
- 你在调试非标准链路

#### 4. 冷启动回看窗口改用更清晰的新变量

推荐新变量：

- `INITIAL_SYNC_LOOKBACK_SECOND`

兼容旧变量：

- `TOO_OLD_TIMESTAMP`

优先级如下：

1. `INITIAL_SYNC_LOOKBACK_SECOND`
2. `TOO_OLD_TIMESTAMP`
3. 默认值 `24`（旧变量语义，按秒解释）

这里需要特别注意：

- `TOO_OLD_TIMESTAMP` 是历史遗留命名，含义并不清晰
- 当前兼容逻辑仍按旧语义解析它
- 新配置请统一使用 `INITIAL_SYNC_LOOKBACK_SECOND`

## 不建议继续合并的配置

下面几组变量看起来相近，但职责不同，不建议为了“少几行”而硬合并：

- `ORDER_CONFIRMATION_TIMEOUT_MS` / `ORDER_CONFIRMATION_POLL_MS` / `ORDER_CONFIRMATION_BLOCKS`
    - 分别控制最长等待时间、轮询频率、确认深度，语义不同
- `AUTO_REDEEM_INTERVAL_MS` / `AUTO_REDEEM_MAX_CONDITIONS_PER_RUN`
    - 一个控制执行节奏，一个控制单次批量大小
- `POLY_BUILDER_API_KEY` / `POLY_BUILDER_SECRET` / `POLY_BUILDER_PASSPHRASE`
    - 这是外部鉴权协议要求的三元组，不宜再压成单字段
- `MAX_SLIPPAGE_BPS` / `MAX_ORDER_USDC`
    - 一个管价格容忍度，一个管单笔资金上限，属于两类风控

结论是：

- 可以共享默认值的，已经通过上层变量收口
- 语义独立的，继续拆开更安全，也更便于排查故障

## 最小配置示例

### trace 最小示例

```env
EXECUTION_MODE=trace
USER_ADDRESS=0xYourTargetWalletAddress
MONGO_URI=mongodb://127.0.0.1:27017/polymarket_copytrading_trace

TRACE_ID=paper
TRACE_INITIAL_BALANCE=1000

FETCH_INTERVAL=1
INITIAL_SYNC_LOOKBACK_SECOND=24
RETRY_LIMIT=3
```

### live 最小示例

```env
EXECUTION_MODE=live
USER_ADDRESS=0xYourTargetWalletAddress
MONGO_URI=mongodb://127.0.0.1:27017/polymarket_copytrading_live

PROXY_WALLET=0xYourProxyWalletAddress
PRIVATE_KEY=your_private_key_without_0x

FETCH_INTERVAL=1
INITIAL_SYNC_LOOKBACK_SECOND=24
RETRY_LIMIT=3
MAX_SLIPPAGE_BPS=300
MAX_ORDER_USDC=50
PROCESSING_LEASE_MS=30000
AUTO_REDEEM_ENABLED=true
MARKET_WS_ENABLED=true
```

如果你只是在官方主网环境运行，上面已经足够。只有在需要覆盖基础设施地址或高级调优时，才继续增加下面的变量。

## 详细配置清单

### 一、通用核心配置

| 变量                          | 是否必填 | 默认值 | 说明                                      |
| ----------------------------- | -------- | ------ |-----------------------------------------|
| `EXECUTION_MODE`              | 否       | `live` | `trace` 为模拟执行，`live` 为真实下单。建议先跑 `trace`。 |
| `USER_ADDRESS`                | 是       | 无     | 被跟单的目标钱包地址。                             |
| `MONGO_URI`                   | 是       | 无     | MongoDB 连接串。建议 `trace` 与 `live` 使用不同数据库。 |
| `FETCH_INTERVAL`              | 否       | `1`    | 监视器轮询间隔，单位秒。                            |
| `INITIAL_SYNC_LOOKBACK_SECOND` | 否       | 无     | 冷启动或同步状态缺失时，向前回看多少秒的活动。推荐新配置使用它。        |
| `RETRY_LIMIT`                 | 否       | `3`    | 执行失败后的最大重试次数。                           |

### 二、trace 模式配置

| 变量                    | 是否必填 | 默认值    | 说明                                          |
| ----------------------- | -------- | --------- | --------------------------------------------- |
| `TRACE_ID`              | 否       | `default` | 模拟运行标签，会体现在 trace 集合名和日志中。 |
| `TRACE_INITIAL_BALANCE` | 否       | `1000`    | 模拟初始资金，单位 USDC。                     |

### 三、live 模式核心配置

| 变量                  | 是否必填    | 默认值  | 说明                                                      |
| --------------------- | ----------- | ------- | --------------------------------------------------------- |
| `PROXY_WALLET`        | `live` 必填 | 无      | 实际执行下单的钱包地址。                                  |
| `PRIVATE_KEY`         | `live` 必填 | 无      | 执行钱包私钥，必须是 64 位十六进制字符串，且不要带 `0x`。 |
| `MAX_SLIPPAGE_BPS`    | 否          | `300`   | 最大允许滑点，单位 bps。                                  |
| `MAX_ORDER_USDC`      | 否          | `0`     | 单笔最大下单金额；`0` 表示不限制。                        |
| `PROCESSING_LEASE_MS` | 否          | `30000` | 待执行记录的处理租约超时。                                |
| `AUTO_REDEEM_ENABLED` | 否          | `true`  | 是否启用空闲轮次自动结算回收。                            |
| `MARKET_WS_ENABLED`   | 否          | `true`  | 是否启用市场 WebSocket 盘口缓存。                         |

### 四、基础设施覆盖配置

| 变量                              | 是否必填 | 默认值                                          | 说明                                                           |
| --------------------------------- | -------- | ----------------------------------------------- | -------------------------------------------------------------- |
| `CLOB_HTTP_URL`                   | 否       | `https://clob.polymarket.com`                   | Polymarket CLOB HTTP 地址。                                    |
| `POLYMARKET_WS_BASE_URL`          | 否       | `wss://ws-subscriptions-clob.polymarket.com/ws` | 共享 WebSocket 基础地址，系统会自动拼接 `/market` 与 `/user`。 |
| `CLOB_WS_URL`                     | 否       | 派生自 `POLYMARKET_WS_BASE_URL`                 | 市场通道 WebSocket；仅在需要单独覆盖市场通道时设置。           |
| `USER_WS_URL`                     | 否       | 派生自 `POLYMARKET_WS_BASE_URL`                 | 用户通道 WebSocket；仅在需要单独覆盖用户通道时设置。           |
| `RPC_URL`                         | 否       | `https://polygon.drpc.org`                      | Polygon RPC，用于读取余额和发起回收。                          |
| `USDC_CONTRACT_ADDRESS`           | 否       | Polygon USDC 官方地址                           | 读取余额和结算回收所用的抵押资产地址。                         |
| `POLYMARKET_RELAYER_URL`          | 否       | `https://relayer-v2.polymarket.com`             | 自动回收使用的 relayer 地址。                                  |
| `POLYMARKET_RELAYER_TX_TYPE`      | 否       | `SAFE`                                          | 自动回收交易类型，可选 `SAFE` / `PROXY`。                      |
| `POLYMARKET_CTF_CONTRACT_ADDRESS` | 否       | 官方 CTF 地址                                   | 自动回收调用的合约地址。                                       |

### 五、同步、确认与缓存参数

| 变量                            | 是否必填 | 默认值     | 说明                                            |
| ------------------------------- | -------- | ---------- | ----------------------------------------------- |
| `ACTIVITY_SYNC_LIMIT`           | 否       | `500`      | 单次活动接口分页大小。                          |
| `ACTIVITY_SYNC_OVERLAP_MS`      | 否       | `30000`    | 增量同步窗口重叠时间，避免边界漏单。            |
| `SNAPSHOT_STALE_AFTER_MS`       | 否       | `30000`    | 快照超过该时间后会被视为陈旧。                  |
| `ORDER_CONFIRMATION_TIMEOUT_MS` | 否       | `45000`    | 等待订单最终确认的最长时间。                    |
| `ORDER_CONFIRMATION_POLL_MS`    | 否       | `2000`     | 回退到链上轮询确认时的轮询间隔。                |
| `ORDER_CONFIRMATION_BLOCKS`     | 否       | `2`        | 链上确认所需的最小区块数。                      |
| `POLYMARKET_WS_RECONNECT_MS`    | 否       | `1000`     | 市场通道与用户通道共享的重连间隔。              |
| `MARKET_WS_RECONNECT_MS`        | 否       | 继承共享值 | 市场通道专属重连间隔。                          |
| `USER_WS_RECONNECT_MS`          | 否       | 继承共享值 | 用户通道专属重连间隔。                          |
| `MARKET_CACHE_TTL_MS`           | 否       | `3000`     | 市场快照缓存有效期。                            |
| `MARKET_WS_SNAPSHOT_WAIT_MS`    | 否       | `750`      | 请求市场快照前，等待 WebSocket 首次补齐的时间。 |

### 六、自动回收与 builder 鉴权

| 变量                                 | 是否必填 | 默认值  | 说明                               |
| ------------------------------------ | -------- | ------- | ---------------------------------- |
| `AUTO_REDEEM_INTERVAL_MS`            | 否       | `30000` | 自动回收扫描间隔。                 |
| `AUTO_REDEEM_MAX_CONDITIONS_PER_RUN` | 否       | `8`     | 单次回收最多处理多少个 condition。 |
| `POLY_BUILDER_API_KEY`               | 否       | 空      | builder 鉴权 key。                 |
| `POLY_BUILDER_SECRET`                | 否       | 空      | builder 鉴权 secret。              |
| `POLY_BUILDER_PASSPHRASE`            | 否       | 空      | builder 鉴权 passphrase。          |

只有当 relayer 或 builder 网关要求额外签名认证时，才需要填写这组三元组。

### 七、兼容变量

| 变量                     | 状态     | 说明                                                 |
| ------------------------ | -------- | ---------------------------------------------------- |
| `TOO_OLD_TIMESTAMP`      | 兼容保留 | 历史变量，建议迁移到 `INITIAL_SYNC_LOOKBACK_SECOND`。 |
| `CLOB_WS_URL`            | 继续支持 | 如需单独覆盖市场通道地址，可继续使用。               |
| `USER_WS_URL`            | 继续支持 | 如需单独覆盖用户通道地址，可继续使用。               |
| `MARKET_WS_RECONNECT_MS` | 继续支持 | 如需给市场通道单独设置重连间隔，可继续使用。         |
| `USER_WS_RECONNECT_MS`   | 继续支持 | 如需给用户通道单独设置重连间隔，可继续使用。         |

## 推荐维护方式

推荐把配置分成两层：

1. `.env` 只放最小必填项和常调项
2. 高级覆盖项只在明确需要时再追加

建议长期保留在 `.env` 里的通常只有这些：

- `EXECUTION_MODE`
- `USER_ADDRESS`
- `MONGO_URI`
- `PROXY_WALLET`
- `PRIVATE_KEY`
- `FETCH_INTERVAL`
- `INITIAL_SYNC_LOOKBACK_SECOND`
- `RETRY_LIMIT`
- `MAX_SLIPPAGE_BPS`
- `MAX_ORDER_USDC`

这样后续切钱包、切模式、调风控时，主要改动仍集中在核心配置，不会把基础设施细节和历史兼容项一起带进日常维护。

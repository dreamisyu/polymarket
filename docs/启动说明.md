# 启动说明

## 目标

本文说明如何在隔离环境中启动两种模式：

- `trace`：本地模拟跟单，只记账，不发真实订单
- `live`：真实交易，调用 Polymarket 下单

若需要先理解三段执行链路与推荐参数，请先阅读 [核心跟单流程与推荐配置](./核心跟单流程与推荐配置.md)。

建议优先使用 Node.js 20 LTS。若需要同时运行两种模式，优先使用两个独立工作目录或两个容器；不要让它们共用同一个 `.env` 文件。

## 隔离原则

1. 使用不同配置文件：`trace` 使用 `.env.trace`，`live` 使用 `.env.live`
2. 使用不同 MongoDB 数据库：
    - `mongodb://127.0.0.1:27017/polymarket_copytrading_trace`
    - `mongodb://127.0.0.1:27017/polymarket_copytrading_live`
3. 真实模式使用独立代理钱包，禁止与测试钱包混用
4. 若要并行启动，使用两个目录，例如：
    - `~/bot-trace`
    - `~/bot-live`

## 通用准备

```bash
npm install
```

准备两个环境文件，推荐从 `.env-example` 复制：

```bash
cp .env-example .env.paper
cp .env-example .env.live
```

## 启动模拟模式

`.env.paper` 最小示例：

```env
EXECUTION_MODE=trace
TRACE_ID=paper
TRACE_INITIAL_BALANCE=1000
USER_ADDRESS=0xYourTargetWalletAddress
MONGO_URI=mongodb://127.0.0.1:27017/polymarket_copytrading_trace
FETCH_INTERVAL=1
TOO_OLD_TIMESTAMP=24
RETRY_LIMIT=3
```

单目录运行：

```bash
cp .env.paper .env
npm run dev
```

启动后会写入独立集合：

- `trace_portfolios_*`
- `trace_positions_*`
- `trace_executions_*`

## 启动真实交易模式

`.env.live` 关键示例：

```env
EXECUTION_MODE=live
USER_ADDRESS=0xYourTargetWalletAddress
PROXY_WALLET=0xYourProxyWalletAddress
PRIVATE_KEY=your_private_key_without_0x
CLOB_HTTP_URL=https://clob.polymarket.com
CLOB_WS_URL=wss://clob-ws.polymarket.com
MONGO_URI=mongodb://127.0.0.1:27017/polymarket_copytrading_live
RPC_URL=https://polygon-rpc.com
USDC_CONTRACT_ADDRESS=0x2791Bca1f2de4661ED88A30C99A7a9449Aa84174
FETCH_INTERVAL=1
TOO_OLD_TIMESTAMP=24
RETRY_LIMIT=3
```

启动前先做一次本地校验：

```bash
cp .env.live .env
node validate-bot.js
npm run dev
```

## 并行运行建议

若要同时观察 `trace` 与 `live`，建议复制两个工作目录，各自放一份 `.env`，分别连接不同 MongoDB 数据库。这样最稳妥，不会出现配置串用、日志混杂或误下单。

# 启动说明

## 目标

本文说明如何在隔离环境中启动两种模式：

- `trace`：本地模拟跟单，只记账，不发真实订单
- `live`：真实交易，调用 Polymarket 下单

若需要先理解 `.env` 参数，请阅读 [环境参数说明](./环境参数说明.md)。
若需要先理解运行流、代码树和领域职责，请阅读 [代码流程与架构](./代码流程与架构.md)。

建议优先使用 Node.js 20 LTS。若需要同时运行两种模式，优先使用两个独立工作目录或两个容器；不要让它们共用同一个 `.env` 文件。

## 隔离原则

1. 使用不同配置文件：`trace` 使用 `.env.trace`，`live` 使用 `.env.live`
2. 使用不同 MongoDB 数据库：
    - `mongodb://127.0.0.1:27017/polymarket_copytrading_trace`
    - `mongodb://127.0.0.1:27017/polymarket_copytrading_live`
3. 真实模式使用独立代理钱包，禁止与测试钱包混用
4. 若要并行启动，使用两个目录，例如：
    - `~/bot-trace`
    - `~/bot-live`

## 通用准备

```bash
npm install
```

准备两个环境文件，推荐从 `.env-example` 复制：

```bash
cp .env-example .env.paper
cp .env-example .env.live
```

## 启动模拟模式

`.env.paper` 最小示例：

```env
EXECUTION_MODE=trace
TRACE_ID=paper
TRACE_INITIAL_BALANCE=90
TRACE_SOURCE_LOOKBACK_MS=0
USER_ADDRESS=0xYourTargetWalletAddress
MONGO_URI=mongodb://127.0.0.1:27017/polymarket_copytrading_trace
FETCH_INTERVAL=1
INITIAL_SYNC_LOOKBACK_SECOND=24
RETRY_LIMIT=3
BUY_SIZING_MODE=condition_pair_overlay
SIGNAL_BUFFER_MS=5000
SIGNAL_IGNORE_BUY_BELOW_USDC=0.25
FOLLOW_MAX_OPEN_POSITIONS=8
FOLLOW_MAX_ACTIVE_EXPOSURE_USDC=18
FOLLOW_MARKET_SCOPE=crypto_updown_5m
FOLLOW_POSITION_DUST_USDC=0.5
PAIR_OVERLAY_MIN_BUY_USDC=1.2
PAIR_LEADER_MIN_SOURCE_USDC=10
PAIR_LEADER_MIN_SOURCE_COUNT=3
PAIR_LEADER_MIN_SHARE=0.65
PAIR_LEADER_MIN_EDGE_USDC=4
PAIR_STRONG_SOURCE_USDC=18
PAIR_STRONG_SOURCE_COUNT=5
PAIR_STRONG_MIN_SHARE=0.70
PAIR_STRONG_MIN_EDGE_USDC=8
PAIR_LEADER_TICKET_USDC=1.2
PAIR_STRONG_TICKET_USDC=1.8
PAIR_HEDGE_TICKET_USDC=1.2
PAIR_HEDGE_PRICE_SUM_MAX=0.96
PAIR_MAX_ACTIONS_PER_CONDITION=2
```

单目录运行：

```bash
cp .env.paper .env
npm run dev
```

启动后会写入独立集合：

- `trace_portfolios_*`
- `trace_positions_*`
- `trace_executions_*`
- `copy_execution_batches_*_trace_*`

`TRACE_SOURCE_LOOKBACK_MS=0` 表示新的 `trace` 只消费启动后的新源成交，不回放历史待处理活动；对高频 `5min up/down` 场景更适合直接观察盘中跟单效果。
`BUY_SIZING_MODE=condition_pair_overlay` 适合 `90u` 这类小资金去跟会在同一 `condition` 双边建仓的大钱包：系统先按 `5s` 累计同一 `condition` 两个 outcome 的源买单，再判断哪一边是 `leader`。只有当主方向满足 `leaderSourceUsdc / leaderShare / edge` 门槛时才入首边，随后仅在两边可买价之和 `<= 0.96` 时补一笔保守型 `hedge overlay`。
`PAIR_OVERLAY_MIN_BUY_USDC=1.2` 是这套策略的最小有效买入金额，刻意高于 Polymarket 的 `>1 USDC` 平台门槛，避免卡在边界值被拒单。
`FOLLOW_MARKET_SCOPE=crypto_updown_5m` 会把策略范围限制在 `btc-updown-5m` / `eth-updown-5m`，不会把同钱包里的 `15m` 或其他题材市场一起跟进来。
`FOLLOW_POSITION_DUST_USDC=0.5` 会在统计 open positions 上限时忽略策略范围外、已可回收或市值低于 `0.5u` 的仓位，避免历史碎仓长期占坑。

## 启动真实交易模式

`.env.live` 关键示例：

```env
EXECUTION_MODE=live
USER_ADDRESS=0xYourTargetWalletAddress
PROXY_WALLET=0xYourProxyWalletAddress
PRIVATE_KEY=your_private_key_without_0x
MONGO_URI=mongodb://127.0.0.1:27017/polymarket_copytrading_live
FETCH_INTERVAL=1
INITIAL_SYNC_LOOKBACK_SECOND=24
RETRY_LIMIT=3
MAX_SLIPPAGE_BPS=300
MAX_ORDER_USDC=50
PROCESSING_LEASE_MS=30000
AUTO_REDEEM_ENABLED=true
POLYMARKET_RELAYER_TX_TYPE=PROXY
MARKET_WS_ENABLED=true
ACTIVITY_ADJACENT_MERGE_WINDOW_MS=15000
BUY_SIZING_MODE=condition_pair_overlay
SIGNAL_BUFFER_MS=5000
SIGNAL_IGNORE_BUY_BELOW_USDC=0.25
FOLLOW_MAX_OPEN_POSITIONS=8
FOLLOW_MAX_ACTIVE_EXPOSURE_USDC=18
FOLLOW_MARKET_SCOPE=crypto_updown_5m
FOLLOW_POSITION_DUST_USDC=0.5
PAIR_OVERLAY_MIN_BUY_USDC=1.2
PAIR_LEADER_MIN_SOURCE_USDC=10
PAIR_LEADER_MIN_SOURCE_COUNT=3
PAIR_LEADER_MIN_SHARE=0.65
PAIR_LEADER_MIN_EDGE_USDC=4
PAIR_STRONG_SOURCE_USDC=18
PAIR_STRONG_SOURCE_COUNT=5
PAIR_STRONG_MIN_SHARE=0.70
PAIR_STRONG_MIN_EDGE_USDC=8
PAIR_LEADER_TICKET_USDC=1.2
PAIR_STRONG_TICKET_USDC=1.8
PAIR_HEDGE_TICKET_USDC=1.2
PAIR_HEDGE_PRICE_SUM_MAX=0.96
PAIR_MAX_ACTIONS_PER_CONDITION=2
BUY_DUST_RESIDUAL_MODE=trim
LIVE_STATE_REFRESH_MS=1000
LIVE_MAX_STALE_SNAPSHOT_MS=5000
LIVE_EXECUTOR_LOOP_INTERVAL_MS=100
LIVE_CONFIRM_TIMEOUT_MS=45000
LIVE_RECONCILE_AFTER_TIMEOUT_MS=3000
LIVE_PERSIST_MAX_QUEUE_SIZE=2000
LIVE_PERSIST_RETRY_MS=1000
```

如果你使用官方默认主网基础设施，上面已经足够；只有在自定义 RPC、WebSocket 网关或 relayer 时，才需要继续补充覆盖变量，详见 [环境参数说明](./环境参数说明.md)。

说明：

- `POLYMARKET_RELAYER_TX_TYPE` 对代理钱包建议设置为 `PROXY`；若误配为 `SAFE`，系统会在启动时探测并自动切换到可用模式。
- 未显式提供 `POLY_BUILDER_*` 时，自动回收会先尝试通过当前 CLOB L2 身份创建临时 builder key；只有这一步失败才会退回无鉴权路径。

启动前先做一次本地校验：

```bash
cp .env.live .env
npm run validate:test
npm run dev
```

若开启自动回收，执行器会在空闲轮次扫描代理钱包的 `redeemable` 仓位，并通过 relayer 自动领取已结算收益，回收到现金余额后继续参与后续跟单。
`SIGNAL_BUFFER_MS=5000`、`PAIR_LEADER_MIN_SOURCE_USDC=10`、`PAIR_STRONG_SOURCE_USDC=18`、`PAIR_HEDGE_PRICE_SUM_MAX=0.96` 这一组参数，是为了让 `90u` 级别的账户不再逐笔按比例跟随大钱包，而是按同一 `condition` 的主边强度和配对价值做 `leader + hedge overlay` 跟单。
`FOLLOW_MARKET_SCOPE=crypto_updown_5m` 用来排除这个目标钱包同时参与的 `btc/eth updown 15m` 市场，避免 `5m` 跟单资金被更长周期头寸占用。
`FOLLOW_POSITION_DUST_USDC=0.5` 用来避免历史碎仓、范围外仓位和已可回收仓位错误挤占新的 `open positions` 额度。
`LIVE_STATE_REFRESH_MS=1000`、`LIVE_EXECUTOR_LOOP_INTERVAL_MS=100`、`LIVE_MAX_STALE_SNAPSHOT_MS=5000` 这一组参数，决定了实盘“内存主状态 + 允许受控 STALE 快照”的默认执行节奏；通常不建议把刷新周期调得更慢，或把 `LIVE_MAX_STALE_SNAPSHOT_MS` 调得明显更大，否则会直接拉高高频市场的跟单延迟与快照漂移风险。
`LIVE_CONFIRM_TIMEOUT_MS=45000` 与 `LIVE_RECONCILE_AFTER_TIMEOUT_MS=3000` 用于控制真实下单后的双通道确认窗口；前者是主确认超时，后者是超时后补偿对账的短窗口。

默认会在控制台输出日志，并持久化到 `logs/bot.log`；如需调整级别或路径，可在 `.env` 中配置 `LOG_LEVEL`、`LOG_FILE_PATH` 等参数。

## 并行运行建议

若要同时观察 `trace` 与 `live`，建议复制两个工作目录，各自放一份 `.env`，分别连接不同 MongoDB 数据库。这样最稳妥，不会出现配置串用、日志混杂或误下单。

# 环境参数说明

## 使用前提

本文只说明 `.env` 参数，不展开代码实现细节。

示例配置基于以下假设：

- 跟单目标主要活跃在加密货币 `5min up/down` 市场
- 源账户交易频率较高，短时间内可能连续拆分多笔买单
- 目标是尽量贴近源账户节奏，同时降低 `< 1 USDC` 买单被平台拒绝的概率

建议先跑 `trace`，观察一段时间后再切换到 `live`。

## 5min up/down 示例基线

### `.env.trace`

```env
EXECUTION_MODE=trace
TRACE_ID=crypto-5m
TRACE_INITIAL_BALANCE=1000

USER_ADDRESS=0xYourTargetWalletAddress
MONGO_URI=mongodb://127.0.0.1:27017/polymarket_copytrading_trace

FETCH_INTERVAL=1
INITIAL_SYNC_LOOKBACK_SECOND=24
RETRY_LIMIT=3
PROCESSING_LEASE_MS=30000

ACTIVITY_SYNC_LIMIT=500
ACTIVITY_SYNC_OVERLAP_MS=30000
SNAPSHOT_STALE_AFTER_MS=15000

MARKET_WS_ENABLED=true
MARKET_CACHE_TTL_MS=1500
POLYMARKET_WS_RECONNECT_MS=1000
MARKET_WS_SNAPSHOT_WAIT_MS=500

ACTIVITY_ADJACENT_MERGE_WINDOW_MS=12000
BUY_MIN_TOP_UP_ENABLED=true
BUY_MIN_TOP_UP_TRIGGER_USDC=0.8
BUY_DUST_RESIDUAL_MODE=trim
```

```env

USER_ADDRESS=0xYourTargetWalletAddress
```

## 参数解析规则

- `CLOB_WS_URL` 与 `USER_WS_URL` 优先级高于 `POLYMARKET_WS_BASE_URL`
- `MARKET_WS_RECONNECT_MS` 与 `USER_WS_RECONNECT_MS` 优先级高于 `POLYMARKET_WS_RECONNECT_MS`
- `INITIAL_SYNC_LOOKBACK_SECOND` 优先级高于兼容变量 `TOO_OLD_TIMESTAMP`
- `CLOB_HTTP_URL`、`RPC_URL`、`POLYMARKET_RELAYER_URL`、`POLYMARKET_CTF_CONTRACT_ADDRESS`、`USDC_CONTRACT_ADDRESS` 未显式设置时，会回退到仓库内置默认值

## 参数清单

### 通用核心配置

| 变量                           | 模式 | 默认值             | 说明                                                    | 5min up/down 示例                                             |
| ------------------------------ | ---- | ------------------ | ------------------------------------------------------- | ------------------------------------------------------------- |
| `EXECUTION_MODE`               | 全局 | `live`             | 运行模式，`trace` 为模拟记账，`live` 为真实下单。       | `trace` 先验证，再切 `live`                                   |
| `USER_ADDRESS`                 | 全局 | 无                 | 被跟单的源钱包地址。                                    | `0xYourTargetWalletAddress`                                   |
| `MONGO_URI`                    | 全局 | 无                 | MongoDB 连接串。建议 `trace` 与 `live` 分库。           | `mongodb://127.0.0.1:27017/polymarket_copytrading_trace/live` |
| `FETCH_INTERVAL`               | 全局 | `1`                | 监视器轮询间隔，单位秒。                                | `1`                                                           |
| `INITIAL_SYNC_LOOKBACK_SECOND` | 全局 | 兼容回退到 `24` 秒 | 冷启动或游标缺失时，向前回看的活动窗口。                | `24`                                                          |
| `RETRY_LIMIT`                  | 全局 | `3`                | 单条源交易或单个批次的最大重试次数。                    | `3`                                                           |
| `PROCESSING_LEASE_MS`          | 全局 | `30000`            | worker 抢占租约超时；用于回收卡住的交易、缓冲区和批次。 | `30000`                                                       |

### trace 运行参数

| 变量                    | 模式    | 默认值    | 说明                                                    | 5min up/down 示例 |
| ----------------------- | ------- | --------- | ------------------------------------------------------- | ----------------- |
| `TRACE_ID`              | `trace` | `default` | trace 命名空间标识，会体现在 trace 集合名与日志标签中。 | `crypto-5m`       |
| `TRACE_INITIAL_BALANCE` | `trace` | `1000`    | 模拟盘初始 USDC 资金。                                  | `1000`            |

### live 交易参数

| 变量               | 模式   | 默认值 | 说明                                                    | 5min up/down 示例             |
| ------------------ | ------ | ------ | ------------------------------------------------------- | ----------------------------- |
| `PROXY_WALLET`     | `live` | 无     | 真实执行下单的钱包地址。                                | `0xYourProxyWalletAddress`    |
| `PRIVATE_KEY`      | `live` | 无     | 执行钱包私钥，必须是 64 位十六进制字符串且不要带 `0x`。 | `your_private_key_without_0x` |
| `MAX_SLIPPAGE_BPS` | `live` | `300`  | 最大允许滑点，单位 bps。                                | `200`                         |
| `MAX_ORDER_USDC`   | `live` | `0`    | 单笔最大下单金额；`0` 表示不限制。                      | `25`                          |

### 买单补齐与碎单控制

| 变量                          | 模式 | 默认值 | 说明                                                                              | 5min up/down 示例 |
| ----------------------------- | ---- | ------ | --------------------------------------------------------------------------------- | ----------------- |
| `BUY_MIN_TOP_UP_ENABLED`      | 全局 | `true` | 是否允许把接近最小门槛的小额买单补齐到 `1 USDC`。                                 | `true`            |
| `BUY_MIN_TOP_UP_TRIGGER_USDC` | 全局 | `0.7`  | 只有累计金额达到该阈值，才允许补齐到 `1 USDC`。                                   | `0.8`             |
| `BUY_DUST_RESIDUAL_MODE`      | 全局 | `trim` | 买单若会留下 `< 1 USDC` 尾单时的处理策略：`trim` 裁剪、`defer` 暂缓、`off` 关闭。 | `trim`            |

监视器会先按“同市场、同方向、相邻仓位”合并源买单；执行器不再维护独立买单缓冲区，只对单条合并后的买单做现金比例定额与 `< 1 USDC` 补齐/跳过判断。

### 市场数据、同步与确认

| 变量                            | 模式   | 默认值     | 说明                                                    | 5min up/down 示例 |
| ------------------------------- | ------ | ---------- | ------------------------------------------------------- | ----------------- |
| `ACTIVITY_SYNC_LIMIT`           | 全局   | `500`      | 活动接口单次分页大小。                                  | `500`             |
| `ACTIVITY_SYNC_OVERLAP_MS`      | 全局   | `30000`    | 增量同步窗口重叠时间，避免边界漏单。                    | `30000`           |
| `SNAPSHOT_STALE_AFTER_MS`       | 全局   | `30000`    | 源账户余额/仓位快照超过该时间后视为陈旧。               | `15000`           |
| `ORDER_CONFIRMATION_TIMEOUT_MS` | `live` | `45000`    | 等待订单最终确认的最长时间。                            | `45000`           |
| `ORDER_CONFIRMATION_POLL_MS`    | `live` | `2000`     | 回退到链上轮询确认时的轮询间隔。                        | `2000`            |
| `ORDER_CONFIRMATION_BLOCKS`     | `live` | `2`        | 链上确认要求的最小区块数。                              | `2`               |
| `MARKET_WS_ENABLED`             | 全局   | `true`     | 是否启用市场 WebSocket 盘口缓存；关闭后退回 HTTP 拉单。 | `true`            |
| `MARKET_CACHE_TTL_MS`           | 全局   | `3000`     | 市场快照缓存有效期。                                    | `1500`            |
| `POLYMARKET_WS_RECONNECT_MS`    | 全局   | `1000`     | 市场通道与用户通道共享的默认重连间隔。                  | `1000`            |
| `MARKET_WS_RECONNECT_MS`        | 全局   | 继承共享值 | 市场通道专属重连间隔。                                  | 通常不单独设置    |
| `USER_WS_RECONNECT_MS`          | `live` | 继承共享值 | 用户通道专属重连间隔。                                  | 通常不单独设置    |
| `MARKET_WS_SNAPSHOT_WAIT_MS`    | 全局   | `750`      | 订阅盘口后，等待首次 WebSocket 快照补齐的时间。         | `500`             |

| ------------------------------------ | ------ | ----------------------------------------------- | ----------------------------------------------------- | ----------------- |

| ------------------- | -------- | ----------------------------------------------------- | ----------------- |

## 调参优先级

若目标市场主要是加密货币 `5min up/down`，优先观察以下几组参数：

1. `MAX_ORDER_USDC` 与 `MAX_SLIPPAGE_BPS`
2. `ACTIVITY_ADJACENT_MERGE_WINDOW_MS`、`BUY_MIN_TOP_UP_TRIGGER_USDC`
3. `BUY_DUST_RESIDUAL_MODE`、`SNAPSHOT_STALE_AFTER_MS`
4. `MARKET_CACHE_TTL_MS`

通常先在 `trace` 中把“跟上率、尾单跳过率、权益波动”看稳定，再把同一组参数迁移到 `live`。

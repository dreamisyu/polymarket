# 环境参数说明

## 使用前提

本文只说明 `.env` 参数，不展开代码实现细节。

示例配置基于以下假设：

- 跟单目标主要活跃在加密货币 `5min up/down` 市场
- 源账户交易频率较高，短时间内可能连续拆分多笔买单
- 目标是尽量贴近源账户节奏，同时降低 `< 1 USDC` 买单被平台拒绝的概率

建议先跑 `trace`，观察一段时间后再切换到 `live`。

## 参数解析规则

- `CLOB_WS_URL` 与 `USER_WS_URL` 优先级高于 `POLYMARKET_WS_BASE_URL`
- `MARKET_WS_RECONNECT_MS` 与 `USER_WS_RECONNECT_MS` 优先级高于 `POLYMARKET_WS_RECONNECT_MS`
- `CLOB_HTTP_URL`、`RPC_URL`、`POLYMARKET_RELAYER_URL`、`POLYMARKET_CTF_CONTRACT_ADDRESS`、`USDC_CONTRACT_ADDRESS` 未显式设置时，会回退到仓库内置默认值

| 变量                               | 模式 | 默认值  | 说明                                                                                                  | 5min up/down 示例     |
| ---------------------------------- | ---- | ------- | ----------------------------------------------------------------------------------------------------- | --------------------- |
| `BUY_SIZING_MODE`                  | 全局 | `ratio` | 买入定额策略：`ratio` 纯比例，`first_entry_ticket` 为首单定额，`signal_fixed_ticket` 为信号固定票据。 | `signal_fixed_ticket` |
| `SIGNAL_BUFFER_MS`                 | 全局 | `15000` | `signal_fixed_ticket` 的信号窗口；同一 outcome 的源买单会先在窗口内累计，再判断是否触发固定票据。     | `15000`               |
| `SIGNAL_IGNORE_BUY_BELOW_USDC`     | 全局 | `0.25`  | 单笔源买单低于该阈值时直接忽略，不进入信号累计。                                                      | `0.25`                |
| `SIGNAL_WEAK_SOURCE_BUY_USDC`      | 全局 | `18`    | 弱信号的累计源买单金额阈值；仅首枪允许触发。                                                          | `18`                  |
| `SIGNAL_WEAK_SOURCE_BUY_COUNT`     | 全局 | `12`    | 弱信号的累计源买单笔数阈值；仅首枪允许触发。                                                          | `12`                  |
| `SIGNAL_MIN_SOURCE_BUY_USDC`       | 全局 | `22`    | 普通信号的累计源买单金额阈值。                                                                        | `22`                  |
| `SIGNAL_MIN_SOURCE_BUY_COUNT`      | 全局 | `10`    | 普通信号的累计源买单笔数阈值。                                                                        | `10`                  |
| `SIGNAL_STRONG_SOURCE_BUY_USDC`    | 全局 | `34`    | 强信号的累计源买单金额阈值。                                                                          | `34`                  |
| `SIGNAL_STRONG_SOURCE_BUY_COUNT`   | 全局 | `14`    | 强信号的累计源买单笔数阈值。                                                                          | `14`                  |
| `FOLLOW_WEAK_TICKET_USDC`          | 全局 | `1`     | 弱信号触发后的固定跟单票据；只用于首枪。                                                              | `1.0`                 |
| `FOLLOW_FIXED_TICKET_USDC`         | 全局 | `1.5`   | 普通信号触发后的固定跟单票据。                                                                        | `1.5`                 |
| `FOLLOW_STRONG_TICKET_USDC`        | 全局 | `2.5`   | 强信号触发后的固定跟单票据。                                                                          | `2.5`                 |
| `FOLLOW_MAX_OPEN_POSITIONS`        | 全局 | `8`     | 固定票据策略允许同时持有的最大 open position 数。                                                     | `8`                   |
| `FOLLOW_MAX_ACTIVE_EXPOSURE_USDC`  | 全局 | `18`    | 固定票据策略允许占用的最大活跃暴露。                                                                  | `18`                  |
| `FOLLOW_MARKET_SCOPE`              | 全局 | `all`   | 固定票据策略的市场白名单；`crypto_updown_5m` 只放行 `btc-updown-5m` / `eth-updown-5m`。               | `crypto_updown_5m`    |
| `FOLLOW_MAX_TICKETS_PER_CONDITION` | 全局 | `2`     | 同一 `condition/outcome` 最多允许跟单次数；默认两枪，第二枪仅允许普通/强信号。                        | `2`                   |
| `BUY_MIN_TOP_UP_ENABLED`           | 全局 | `true`  | 仅在 `ratio / first_entry_ticket` 模式下使用；允许把接近最小门槛的小额买单补齐到 `1 USDC`。           | `true`                |
| `BUY_MIN_TOP_UP_TRIGGER_USDC`      | 全局 | `0.7`   | 仅在 `ratio / first_entry_ticket` 模式下使用；累计金额达到该阈值后，才允许补齐到 `1 USDC`。           | `0.8`                 |
| `BUY_INTENT_BUFFER_MAX_MS`         | 全局 | `2000`  | 仅在 `ratio / first_entry_ticket` 模式下使用；累计 `< 1 USDC` 买单时的最大等待窗口。                  | `2000`                |
| `BUY_FIRST_ENTRY_TICKET_USDC`      | 全局 | `1`     | 仅在 `BUY_SIZING_MODE=first_entry_ticket` 时生效；首笔开仓的固定买入金额。                            | `1`                   |
| `BUY_FIRST_ENTRY_SIGNAL_MIN_USDC`  | 全局 | `0.05`  | 仅在 `BUY_SIZING_MODE=first_entry_ticket` 时生效；比例换算金额达到该阈值后才允许触发首单定额。        | `0.05`                |
| `BUY_BOOTSTRAP_MAX_ACTIVE_RATIO`   | 全局 | `0.15`  | 仅在 `ratio / first_entry_ticket` 模式下使用；限制首单票据与补齐票据占总权益的比例。                  | `0.15`                |
| `BUY_DUST_RESIDUAL_MODE`           | 全局 | `trim`  | 买单若会留下 `< 1 USDC` 尾单时的处理策略：`trim` 裁剪、`defer` 暂缓、`off` 关闭。                     | `trim`                |

对于 `90u` 跟 `0x297f...5103` 这类大钱包高频买入场景，建议优先使用 `signal_fixed_ticket`：系统会先过滤 `< 0.25 USDC` 的源碎单，再按 `15s` 累计窗口判断是否达到 `18 USDC / 12 笔`、`22 USDC / 10 笔`、`34 USDC / 14 笔` 三档阈值。触发后不再按比例缩放，而是直接下 `1.0 USDC`、`1.5 USDC` 或 `2.5 USDC` 固定票据，并用 `FOLLOW_MAX_OPEN_POSITIONS`、`FOLLOW_MAX_ACTIVE_EXPOSURE_USDC`、`FOLLOW_MAX_TICKETS_PER_CONDITION=2` 控制总暴露和单题材跟入次数。对这个地址，额外建议把 `FOLLOW_MARKET_SCOPE` 固定为 `crypto_updown_5m`，因为它会同时交易 `btc/eth updown 15m`，不适合让 `90u` 资金混跟。

| `POLY_BUILDER_API_KEY`    | `live` | 空     | builder 鉴权 key；未设置时自动回收仍可尝试走无鉴权路径。 | 按账户实际填写    |
| `POLY_BUILDER_SECRET`     | `live` | 空     | builder 鉴权 secret。                                    | 按账户实际填写    |
| `POLY_BUILDER_PASSPHRASE` | `live` | 空     | builder 鉴权 passphrase。                                | 按账户实际填写    |

## 调参优先级

若目标市场主要是加密货币 `5min up/down`，优先观察以下几组参数：

1. `MAX_ORDER_USDC` 与 `MAX_SLIPPAGE_BPS`
2. `BUY_SIZING_MODE`、`SIGNAL_BUFFER_MS`、`SIGNAL_MIN_SOURCE_BUY_USDC`、`SIGNAL_MIN_SOURCE_BUY_COUNT`
3. `FOLLOW_FIXED_TICKET_USDC`、`FOLLOW_STRONG_TICKET_USDC`、`FOLLOW_MAX_ACTIVE_EXPOSURE_USDC`
4. `ACTIVITY_ADJACENT_MERGE_WINDOW_MS`、`FOLLOW_MAX_TICKETS_PER_CONDITION`
5. `SNAPSHOT_STALE_AFTER_MS`、`MARKET_CACHE_TTL_MS`

通常先在 `trace` 中把“跟上率、尾单跳过率、权益波动”看稳定，再把同一组参数迁移到 `live`。
